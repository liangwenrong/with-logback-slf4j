二、日志信息之所以重要是因为有用，它记录了应用系统运行过程中的重要信息
我们可以通过这些日志记录，来监控应用的访问，运行性能，报错，排查定位问题，确认数据是否被修改等，
有了记录，遇到问题后可以凭借日志来解决问题和坚定立场

日志格式的配置，和输出什么日志是关键
一般来说需要：
1、记录时间：知道什么时候发生的
2、记录具体位置：知道发生的代码位置，具体在使用什么功能
3、记录原因：知道为什么打印这个日志
4、反过来思考，记录这个信息的目的是什么，需要提供哪些信息才能达到日志的效果


日志的管理
一般来说，日志直接输出到数据库，然后通过查询页面直接查看，这样是非常高效的
日志文件的切割，建议从两个角度
1、从时间角度，一般是每天滚动一份日志文件
2、从严重级别角度，error级别单独打印在一个文件，可以有效的监控错误和排查问题
